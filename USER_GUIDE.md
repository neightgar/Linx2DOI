## Введение

### Что делает Linx2DOI?

Linx2DOI автоматически:
1. Извлекает список статей из Word документа
2. Находит DOI для каждой статьи
3. Получает метаданные из CrossRef
4. Формирует APA цитаты
5. Создает красивый HTML отчет
6. Экспортирует в Mendeley/Zotero

### Кому это нужно?

- **Студенты** - оформление списка литературы
- **Исследователи** - библиография для статей
- **Преподаватели** - проверка студенческих работ
- **Библиотекари** - каталогизация публикаций

## Установка и запуск

### Windows

```cmd
1. Скачать Python с python.org (версия 3.8+)
2. Открыть командную строку
3. cd C:\путь\к\Linx2DOI
4. pip install -r requirements.txt
5. python main.py
```

### macOS

```bash
# Python обычно уже установлен
cd /path/to/Linx2DOI
pip3 install -r requirements.txt
python3 main.py
```

### Linux

```bash
sudo apt install python3 python3-pip  # Debian/Ubuntu
cd /path/to/Linx2DOI
pip3 install -r requirements.txt
python3 main.py
```

## Подготовка документа

### Правильный формат

✅ **Правильно:**
```
1. Smith J. Neural networks in AI https://doi.org/10.1234/example
2. Doe M. Machine learning basics https://pubmed.ncbi.nlm.nih.gov/12345678/
3. Johnson K. Deep learning https://www.researchgate.net/publication/123456
```

❌ **Неправильно:**
```
Smith J. Neural networks...           # Нет номера
• Doe M. Machine learning...          # Маркер вместо номера
Article without URL                    # Нет URL
```

### Поддерживаемые типы URL

| Тип | Пример | Распознавание |
|-----|--------|---------------|
| DOI прямой | `https://doi.org/10.1234/abc` | ✅ Отлично |
| PubMed | `https://pubmed.ncbi.nlm.nih.gov/12345/` | ✅ Отлично |
| PMC | `https://www.ncbi.nlm.nih.gov/pmc/articles/PMC12345/` | ✅ Отлично |
| ResearchGate | `https://www.researchgate.net/publication/123456` | ⚠️ Хорошо |
| Журнал | `https://www.nature.com/articles/nature12345` | ⚠️ Средне |
| Произвольный | `https://example.com/article.html` | ⚠️ Слабо |

### Создание документа

**Microsoft Word:**
1. Создать новый документ
2. Главная → Нумерация (1., 2., 3.,...)
3. Вставить название статьи
4. Добавить URL статьи
5. Сохранить как .docx

**Google Docs:**
1. Создать документ
2. Формат → Маркеры и нумерация → Нумерованный список
3. Добавить статьи
4. Файл → Скачать → Microsoft Word (.docx)

## Анализ цитирований

### Что это?

Linx2DOI v1.7 автоматически анализирует цитирования в вашем документе:

**Распознаваемые форматы:**
- Одиночные: `[1]`, `[25]`, `[100]`
- Диапазоны: `[1-5]`, `[10-15]`
- Комбинированные: `[1, 3, 5-7]`, `[2, 4-6, 8]`

### Как это работает?

**1. Сканирование документа:**
```
При обработке .docx файла программа ищет:
- Все вхождения формата [число]
- Диапазоны [число-число]
- Комбинации [1, 3, 5-7]
```

**2. Пометка записей:**
```
Запись №1 → цитируется → статус ОК (поиск DOI)
Запись №4 → не цитируется → статус ОТСУТСТВУЕТ В ТЕКСТЕ
Запись №7 → цитируется → статус ОК (поиск DOI)
```

**3. Умный пропуск:**
```
Записи со статусом "ОТСУТСТВУЕТ В ТЕКСТЕ":
- Не обрабатываются автоматически (экономия API)
- Отображаются серым цветом в таблице
- Исключаются из финального Word файла
```

### Выборочная обработка

**Для обработки нецитируемой записи:**

**Способ 1 - Галочка:**
```
1. Поставьте галочку в колонке "Выбрать"
2. Нажмите [⚙️ Обработать выбранные]
3. Запись будет обработана независимо от цитирования
```

**Способ 2 - Ручной DOI:**
```
1. Введите DOI в колонку "Ручной DOI"
2. Нажмите [💾 Применить ручные DOI]
3. Галочка не требуется (логика "ИЛИ")
```

### Статусы цитирований

| Статус | Значение | Обработка |
|--------|----------|-----------|
| **Цитируется** | Есть `[N]` в тексте | Автоматический поиск DOI |
| **ОТСУТСТВУЕТ В ТЕКСТЕ** | Нет цитирований | Пропуск (экономия API) |
| **Выбрано** | Галочка в колонке | Принудительная обработка |

### Пример работы

**Документ:**
```
Список литературы:
1. Smith J. Neural networks...
2. Doe M. Machine learning...
3. Johnson K. Deep learning...
4. Brown A. AI ethics...
5. Wilson R. Data science...

Текст диссертации:
"...как показано в [1, 3]..."
```

**Результат обработки:**
```
№1 → цитируется → ОК (DOI найден)
№2 → ОТСУТСТВУЕТ В ТЕКСТЕ → пропуск
№3 → цитируется → ОК (DOI найден)
№4 → ОТСУТСТВУЕТ В ТЕКСТЕ → пропуск
№5 → ОТСУТСТВУЕТ В ТЕКСТЕ → пропуск
```

> 💡 **Совет:** Для обработки записей №2, №4, №5 поставьте галочки или введите DOI вручную

## Ручной ввод DOI

### Когда нужно?

- Статус "НЕТ ДАННЫХ"
- Некорректный DOI найден
- Хотите переопределить автоматический результат

### Как ввести?

1. **Найти строку** с "НЕТ ДАННЫХ"
2. **Кликнуть** в колонку "Ручной DOI"
3. **Ввести DOI**: `10.1234/example`
4. **Нажать** [💾 Применить ручные DOI]

```
Было:
│ 3 │ Failed │ Не найден │ ... │ [      ] │ НЕТ ДАННЫХ │

Вводим:
│ 3 │ Failed │ Не найден │ ... │ [10.123] │ НЕТ ДАННЫХ │

После "Применить":
│ 3 │ Failed │ Article Title │ 10.123 │ [...] │ РУЧНОЙ DOI │
```

### Формат DOI

✅ **Правильно:**
- `10.1234/example`
- `10.1038/nature12345`
- `10.1126/science.abc1234`

❌ **Неправильно:**
- `https://doi.org/10.1234/example` (уберите URL часть)
- `doi:10.1234/example` (уберите префикс)
- `10.1234` (неполный DOI)

### Проверка DOI

**Вручную проверить DOI:**
1. Открыть браузер
2. Перейти на `https://doi.org/10.ВАШИЙ.DOI`
3. Если открылась статья - DOI правильный

### Собрать публикации в Word

```
[📚 Собрать публикации]
```

**Создает docx файл** с отформатированным списком публикаций в стиле APA.

**Что входит:**
- Все записи **кроме** статуса "ОТСУТСТВУЕТ В ТЕКСТЕ"
- Сохраняется исходная нумерация (с пропусками)
- Номера как обычный текст (Word не перенумерует)

**Форматирование записей:**

1. **Записи с данными (ОК/РУЧНОЙ DOI)** → Полная APA цитата
   ```
   1. Smith, J., & Brown, K. (2024). Neural networks in AI. 
      Nature, 615(7950), 123-145. https://doi.org/10.1234/example
   ```

2. **НЕТ ДАННЫХ** → Название с гиперссылкой на URL
   ```
   7. The leaving group ability in CH(3)LG model system
      https://www.researchgate.net/publication/12345
   ```

3. **ДУБЛЬ** → Только текст "ДУБЛЬ № X"
   ```
   8. ДУБЛЬ № 3
   ```

**Пример:**
- Исходные позиции: 1,2,3 (ОК), 4,5,6 (ОТСУТСТВУЕТ), 7 (НЕТ ДАННЫХ), 8 (ДУБЛЬ №3)
- В docx файле: записи **1, 2, 3, 7, 8** с исходной нумерацией

**Использование:**
```
1. Нажать [📚 Собрать публикации]
2. Выбрать место сохранения
3. Ввести имя (например, publications.docx)
4. Открыть в Word
5. Отредактировать при необходимости
```

**Формат файла:**
- Times New Roman, 12pt
- Выравнивание по левому краю
- Гиперссылки синие, подчёркнутые
- Готово для вставки в диссертацию/статью

**Автоматическое исключение:**
- Записи "ОТСУТСТВУЕТ В ТЕКСТЕ" не попадают в файл
- Экономия места и времени на редактирование
- Только реально цитируемые источники

## Часто задаваемые вопросы

### Общие вопросы

**Q: Сколько статей можно обработать за раз?**

A: Технически - без ограничений. Практически:
- До 50 статей - быстро (~2-3 минуты)
- 50-100 статей - средне (~5-10 минут)
- 100+ статей - медленно (зависит от API)

---

**Q: Нужен ли интернет?**

A: Да, обязательно. Программа делает запросы к:
- CrossRef API
- NCBI E-utilities (PubMed/PMC)
- DOI.org

---

**Q: Безопасно ли вводить email?**

A: Да. Email используется только для:
- HTTP заголовка User-Agent в запросах к NCBI
- Соблюдения требований API

Email НЕ отправляется никуда кроме NCBI и не используется для других целей.

---

**Q: Почему некоторые DOI не находятся?**

A: Возможные причины:
1. У статьи нет DOI (старые публикации, препринты)
2. URL не содержит идентификатор (PMID, PMCID, DOI)
3. CrossRef не находит по заголовку
4. API временно недоступен

Решение: Введите DOI вручную.

---

**Q: Что делать с дублями?**

A: Статьи с одинаковым DOI помечаются как "ДУБЛЬ № X":
- Проверьте оригинальный документ
- Возможно, статья упомянута дважды
- Можно удалить дубликат из документа и переобработать

---

**Q: Что означает статус "ОТСУТСТВУЕТ В ТЕКСТЕ"?**

A: Это означает, что запись не процитирована в документе:
- Запись есть в списке литературы
- Но в тексте нет ссылки `[N]` на эту запись
- Обработка пропускается для экономии API

**Что делать:**
- Если запись не нужна - оставьте как есть (исключится из Word файла)
- Если нужна - поставьте галочку или введите DOI вручную

---

### Технические вопросы

**Q: Поддерживается ли .doc (старый формат)?**

A: Нет, только .docx. Конвертируйте:
- Word: Файл → Сохранить как → .docx
- LibreOffice: File → Save As → .docx

---

**Q: Можно ли обработать PDF?**

A: Пока нет. Запланировано в будущих версиях.

---

**Q: Как очистить результаты?**

A: Просто выберите новый файл - результаты очистятся автоматически.

---

**Q: Где хранятся настройки?**

A: В системном реестре (Windows) или файлах настроек (macOS/Linux):
- Windows: `HKEY_CURRENT_USER\Software\Linx2DOI`
- macOS: `~/Library/Preferences/com.Linx2DOI.Linx2DOI.plist`
- Linux: `~/.config/Linx2DOI/Linx2DOI.conf`

---

**Q: Программа зависла, что делать?**

A:
1. Подождите 30-60 секунд (API может быть медленным)
2. Проверьте лог - возможно, обработка идет
3. Если действительно зависла - закройте и откройте заново
4. Данные не потеряются при следующем запуске

## Советы и хитрости

### Ускорение работы

**1. Подготовьте качественные URL:**
- Используйте прямые DOI ссылки (`doi.org/...`)
- PubMed PMID ссылки работают быстрее

**2. Правильный email:**
- Используйте реальный email
- NCBI может приоритизировать корректные запросы

**3. Обрабатывайте партиями:**
- Вместо 200 статей сразу
- Разбейте на 4 файла по 50

### Использование анализа цитирований

**1. Проверка цитирований:**
- После обработки просмотрите статусы
- "ОТСУТСТВУЕТ В ТЕКСТЕ" = запись не цитируется
- Это нормально для вспомогательных списков

**2. Экономия времени:**
- Нецитируемые записи не тратят API запросы
- Обработка проходит быстрее
- Меньше нагрузки на серверы

**3. Выборочная обработка:**
- Для обработки нецитируемой записи:
  - Поставьте галочку ИЛИ
  - Введите DOI вручную
- Логика "ИЛИ" упрощает работу

### Качество результатов

**1. Проверяйте статусы:**
- После обработки просмотрите колонку "Статус"
- Все "НЕТ ДАННЫХ" - кандидаты на ручной ввод

**2. Проверяйте дубли:**
- "ДУБЛЬ" статус может указывать на ошибку в документе
- Или на действительно повторную ссылку

**3. Используйте HTML для проверки:**
- Откройте HTML файл
- Проверьте все ли APA цитаты корректны
- Кликните на заголовки - проверьте что открывается нужная статья

### Экспорт в Mendeley

**1. Группировка:**
- Создайте папку в Mendeley перед импортом
- Импортируйте RIS в эту папку
- Удобно организовывать по проектам

**2. Теги:**
- После импорта добавьте теги в Mendeley
- Например: "Диссертация 2024", "Проект X"

**3. Дубликаты:**
- Mendeley может обнаружить дубликаты
- Проверьте и объедините при необходимости

### Работа с ручными DOI

**1. Поиск DOI:**
- Google: `"название статьи" DOI`
- CrossRef: `https://search.crossref.org/`
- PubMed: найти статью → посмотреть DOI

**2. Пакетный ввод:**
- Введите все DOI в колонку "Ручной DOI"
- Один раз нажмите "Применить ручные DOI"
- Обработаются все сразу

**3. Проверка:**
- После применения проверьте заголовок статьи
- Если появилось название - DOI правильный
- Если "Не найден" - DOI неверный
