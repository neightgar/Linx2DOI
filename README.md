# 📚 Linx2DOI

**DOI Extractor & APA Bibliography Generator**

Профессиональное приложение для автоматического извлечения DOI из документов Word и генерации библиографии в стиле APA.

> **Linx2DOI** — игра слов: "links" (ссылки/референсы) + "DOI"

![Version](https://img.shields.io/badge/version-1.6-blue)
![Python](https://img.shields.io/badge/python-3.8%2B-blue)
![License](https://img.shields.io/badge/license-MIT-green)

---

## ✨ Основные возможности

### 🔍 Автоматический поиск DOI
- **Множество источников**: DOI.org, PubMed, PMC, ResearchGate, CrossRef
- **Интеллектуальный парсинг**: Распознавание PMID, PMCID, прямых ссылок, гиперссылок Word
- **Улучшенное сопоставление**: Двунаправленная проверка + similarity matching (75%)
- **Fallback стратегии**: Поиск по заголовку если URL не содержит идентификатор
- **Обнаружение дублей**: Автоматическое выявление повторяющихся DOI
- **Поддержка Word нумерации**: Автоматическая нумерация Word распознается

### 📝 Библиография
- **APA 7-е издание**: Полностью соответствует стандарту
- **Форматирование авторов**: Правильная обработка списков авторов
- **Метаданные**: Название журнала, том, выпуск, страницы, год

### 📤 Экспорт
- **RIS формат**: Для Mendeley, Zotero, EndNote, Papers (дубли исключены)
- **HTML отчеты**: Кликабельные заголовки с оригинальными URL из документа
- **Статистика**: Автоматический подсчет результатов и дублей
- **Fallback заголовки**: Оригинальные названия для записей без метаданных

### 🎨 Интерфейс
- **Светлая/Темная тема**: Переключение одним кликом, улучшенная контрастность
- **Видимый прогресс**: Темный текст прогрессбара в светлой теме (виден всегда)
- **Удобная таблица**: Редактирование, сортировка, темный текст при выборе
- **Вкладки**: Разделение результатов и лога
- **Прогресс-бар**: Отслеживание процесса обработки (300×22px с процентами)

### 🛠️ Дополнительно
- **Ручной ввод DOI**: Для статей без автоматического обнаружения
- **Сохранение настроек**: Email, тема, последний файл
- **Кэширование**: Быстрая повторная обработка

## 🚀 Использование

### 1. Подготовка документа

Создайте `.docx` файл с нумерованным списком статей:

```
1. Article Title 1..... https://pubmed.ncbi.nlm.nih.gov/12345678/
2. Another Article..... https://doi.org/10.1234/example
3. ResearchGate Link... https://www.researchgate.net/publication/123456789
```

**Формат:**
- Нумерованный список (1., 2., 3., ...)
- Название статьи
- URL на статью (любой формат)

### 2. Запуск приложения

```bash
python main.py
```

### 3. Workflow

#### Шаг 1: Email
```
┌─────────────────────────────────┐
│ Email для NCBI API:             │
│ [example@gmail.com]             │
└─────────────────────────────────┘
```
> Требуется для доступа к PubMed/PMC

#### Шаг 2: Выбор файла
```
[📁 Выбрать] → Выбрать .docx файл
```

#### Шаг 3: Обработка
```
[🚀 Запустить обработку]
```
- Прогресс-бар показывает процесс
- Лог отображает детали каждой статьи
- Результаты появляются в таблице

#### Шаг 4: Ручные DOI (опционально)
Для статей с статусом "НЕТ ДАННЫХ":
```
1. Введите DOI в колонку "Ручной DOI"
2. Нажмите [💾 Применить ручные DOI]
```

#### Шаг 5: Экспорт
```
[📄 Открыть результат]   # HTML файл
[📥 Экспорт в RIS]        # Для Mendeley
```

## 🐛 Устранение неполадок

### Частые проблемы

**1. "Не найден DOI"**
```
Причины:
- У статьи действительно нет DOI
- URL некорректный
- API временно недоступен

Решение:
- Введите DOI вручную в колонку "Ручной DOI"
- Проверьте подключение к интернету
```

**2. "Email не валиден"**
```
Причина:
- Неправильный формат email

Решение:
- Используйте формат: example@domain.com
```

**3. "Не распознаются статьи"**
```
Причина:
- Неправильный формат списка

Решение:
- Используйте нумерованный список: "1. Название... URL"
- Каждая статья - отдельный пункт
```

**4. "Долго обрабатывается"**
```
Причина:
- Большой список статей
- Rate limiting API

Решение:
- Подождите (есть прогресс-бар)
- API имеют задержки между запросами
```

## 🔬 Технические детали

### Стратегии поиска DOI (по приоритету)

1. **Ручной DOI** - наивысший приоритет
2. **Regex в тексте** - поиск паттерна `10.xxxx/xxxx`
3. **DOI URL** - парсинг `doi.org` ссылок
4. **PubMed PMID** - извлечение PMID → API запрос
5. **PMC PMCID** - извлечение PMCID → API запрос
6. **ResearchGate** - извлечение заголовка из URL → CrossRef поиск с улучшенным сопоставлением
7. **CrossRef fallback** - поиск по заголовку статьи (двунаправленная проверка + similarity 75%)

### Парсинг документов

- **Word нумерация**: Поддержка встроенной автоматической нумерации Word (`numPr`)
- **Гиперссылки**: Приоритетное извлечение полных URL из гиперссылок документа
- **Разбитые URL**: Автоматическое объединение URL, разделенных абзацами
- **URL с пробелами**: Корректная обработка и очистка пробелов в ссылках

### Rate Limiting

```python
MAX_RETRIES = 3           # Максимум попыток
REQUEST_TIMEOUT = 10      # Таймаут запроса (сек)
RATE_LIMIT_DELAY = 0.5    # Задержка между запросами (сек)
```

### Многопоточность

- **UI Thread** - главный поток (PyQt6)
- **Worker Thread** - фоновая обработка (QThread)
- **Signals** - безопасная коммуникация между потоками
