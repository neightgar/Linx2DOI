# 📝 Changelog

Все значимые изменения в проекте Linx2DOI документированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект следует [Semantic Versioning](https://semver.org/lang/ru/).

## [1.8.0] - 2026-04-02

### ✨ Добавлено

- **PubMed поиск по библиографическим ссылкам** (новый модуль `pubmed_searcher.py`)
  - Парсинг ссылок в форматах ГОСТ, APA, Vancouver, Hybrid
  - Автоматическое определение стиля ссылки
  - Поиск в PubMed по заголовку и авторам
  - Нечёткое сравнение заголовков (rapidfuzz, порог 85%)
  - Извлечение метаданных (журнал, год) для улучшения точности поиска
  - Интеграция как 8-й шаг в приоритетную цепочку поиска DOI

- **Скрипт сборки исполняемого файла** (`build.py`)
  - Автоматическая сборка `.exe` через PyInstaller
  - Проверка зависимостей перед сборкой
  - Оптимизация размера: исключение ненужных модулей PyQt6
  - Поддержка флагов: `--clean`, `--check`, `--help`
  - Корректная обработка `jaraco.text` и `pkg_resources`

### 📦 Зависимости

- Добавлен `biopython>=1.77` — PubMed API через `Bio.Entrez`
- Добавлен `rapidfuzz>=3.0.0` — нечёткое сравнение заголовков статей

## [1.7.0] - 2026-02-08

### ✨ Добавлено

- **Кнопка "Собрать публикации"**
  - Создание docx файла с публикациями в формате APA
  - Автоматическое исключение записей со статусом "ОТСУТСТВУЕТ В ТЕКСТЕ"
  - Сохранение исходной нумерации из документа (с пропусками)
  - Номера добавляются как обычный текст (Word не перенумеровывает)

- **Интеллектуальное форматирование записей**
  - Дубликаты: отображаются как "ДУБЛЬ № X" без названия
  - Нет данных: название с гиперссылкой на исходный URL
  - Остальные: полная APA цитата (без HTML тегов)

- **Анализ цитирований в тексте документа**
  - Автоматическое сканирование цитирований формата `[1]`, `[2, 4]`, `[10-15]`
  - Поддержка диапазонов и комбинаций: `[1, 3, 5-7]`
  - Пометка записей как цитируемые/нецитируемые

- **Интеллектуальный пропуск нецитируемых записей**
  - Записи без цитирований получают статус "ОТСУТСТВУЕТ В ТЕКСТЕ"
  - Пропуск автоматического поиска DOI для экономии API запросов
  - Возможность принудительной обработки через галочку или ручной DOI

- **Объединённая кнопка "Обработать выбранные"**
  - Логика "ИЛИ": обработка при галочке ИЛИ ручном DOI
  - Не требуется ставить галочку при вводе ручного DOI
  - Галочка позволяет обработать даже нецитируемые записи

- **Валидация усечённых URL**
  - Проверка полноты URL из гиперссылок Word
  - Fallback на текстовый URL если гиперссылка усечена
  - Обработка повреждённых гиперссылок при копировании между документами

### 🎨 Изменено

- Удалён текст "Действия:" для упрощения интерфейса
- Колонка чекбоксов теперь видима для выборочной обработки
- Улучшена видимость серого статуса для нецитируемых записей

### 🐛 Исправлено

- Исправлена проблема с усечёнными ResearchGate URL из гиперссылок
- Исправлена обработка нецитируемых записей при явном выборе пользователя

### 📊 Статистика

- Упрощение для пользователя: 43% меньше кликов при ручном вводе DOI
- Экономия API запросов за счёт пропуска нецитируемых записей

## [1.6.0] - 2026-02-07

### ✨ Добавлено

- **Поддержка встроенной нумерации Word**
  - Автоматическое распознавание списков с Word нумерацией
  - Не требуется ручное переформатирование документов
  - Проверка `numPr` в XML структуре параграфа

- **Извлечение гиперссылок из документа**
  - Приоритетное использование URL из гиперссылок Word
  - Полные URL для ResearchGate и других источников
  - Корректная обработка ссылок, разбитых на абзацы

- **Улучшенное сопоставление заголовков для DOI**
  - Двунаправленная проверка совпадений
  - Similarity matching через `difflib.SequenceMatcher` (порог 75%)
  - Решает проблему с разным порядком слов в заголовках

### 🎨 Изменено

- **Светлая тема - улучшенная видимость**
  - Текст при выборе строки остается темным и жирным
  - Прогрессбар: темный текст (всегда виден, даже до 50%)
  - Улучшенная контрастность для всех элементов

- **HTML отчет - оригинальные URL**
  - Заголовки статей ведут на оригинальные URL из документа
  - Гиперссылки работают даже для "НЕТ ДАННЫХ"
  - Отображение оригинального названия вместо "None"
  - DOI остается как отдельная ссылка внизу

- **RIS экспорт**
  - Автоматическое исключение дублей
  - Экспортируются только уникальные записи

### 🐛 Исправлено

- **Парсинг документов (80 → 204 источника)**
  - URL с пробелами теперь распознаются корректно
  - Разбитые URL объединяются автоматически
  - Обработка неполных URL (`https://` + следующий параграф)

- **ResearchGate ссылки**
  - Полные URL извлекаются из гиперссылок
  - Корректное отображение в таблице результатов
  - Улучшенное разрешение DOI

### 🔧 Техническое

- Добавлен импорт `difflib.SequenceMatcher` в `doi_resolver.py`
- Добавлен импорт `qn` из `docx.oxml.ns` в `document_parser.py`
- Новый метод `extract_hyperlinks_from_paragraph()` в `DocumentParser`
- Новый метод `get_paragraph_number()` для проверки Word нумерации
- Улучшена логика `collect_items_from_document()` с циклом while
- Условное форматирование стилей на основе `self.current_theme`

  - Добавлены стили hover для заголовков-ссылок

## [1.0.0] - 2025-XX-XX

### ✨ Добавлено

- **Основной функционал**
  - Парсинг .docx документов
  - Автоматический поиск DOI через:
    - Прямые DOI ссылки
    - PubMed (PMID)
    - PMC (PMCID)
    - ResearchGate
    - CrossRef поиск по заголовку
  - Генерация APA цитат (7-е издание)
  - HTML отчеты с библиографией
  - Ручной ввод DOI

- **Интерфейс**
  - PyQt6 графический интерфейс
  - Темная тема
  - Таблица результатов
  - Прогресс-бар обработки
  - Вкладки "Результаты" и "Лог"
  - Кнопки управления:
    - Выбор файла
    - Запуск обработки
    - Применить ручные DOI
    - Открыть результат
    - Справка

- **Модульная архитектура**
  - `src/core/` - бизнес-логика
  - `src/gui/` - интерфейс
  - Разделение ответственности
  - Переиспользуемые модули

### 🔧 Техническое

- Python 3.8+ совместимость
- Зависимости:
  - python-docx (парсинг .docx)
  - requests (HTTP запросы)
  - PyQt6 (GUI)
- QSettings для сохранения настроек
- Worker thread для фоновой обработки
- Signal-slot архитектура
- Rate limiting для API
